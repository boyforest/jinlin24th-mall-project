# 金霖二十四养 · 四周回溯训练方案

> 适用：软件工程大二学生，已有一个月中断，计划融合考研11408备考 + 项目复习 + 学校项目  
> 时长：2026年6月9日 – 7月5日（28天）  
> 工具：IDEA / VS Code / opencode CLI（superpowers 插件已配置）

---

## 一、核心理念

> **你不是"从零学"，你是"以项目为锚点回溯理解"。**  
> 你有现成的全栈代码库——这是我审查过的项目里文档最完整的学生项目之一。利用它。

### 三条学习主线并行

| 主线 | 目标 | 时间 |
|------|------|------|
| **项目回溯** | 逐层搞懂每一行代码为什么这样写，为什么当时AI这样生成 | 2h/天 |
| **11408 考研** | 用项目反哺理论理解，而非死记硬背 | 3h/天 |
| **学校项目** | 每天实践，积累经验（晚间） | 2-3h/天 |

### 三件事的底层统一

```
学校项目（晚间）──→ 生产实际用到的东西
        ↓
项目回溯（白天1）──→ 拆开看"为什么这样用"
        ↓
11408 考研（白天2）──→ "为什么能这样用"的底层原理
```

你不是在做三件事，你是在用三个角度学同一套知识系统。

### 解决"看得懂写不出"的每日微习惯

每天结束时，随机挑一段白天看过的代码，**关掉 IDE**，在纸上画流程图。
- 画得出 → 真懂了
- 画不出 → 明天重点回看

### AI 使用渐进断奶

| 周 | 规则 |
|----|------|
| W1 | AI 可辅助解释代码，可帮你找文件位置 |
| W2 | AI 只能回答"为什么"，不能写新代码 |
| W3 | AI 只能查文档性质的问答，禁用代码生成 |
| W4 | 关闭 AI 辅助，只靠文档和自己 |
