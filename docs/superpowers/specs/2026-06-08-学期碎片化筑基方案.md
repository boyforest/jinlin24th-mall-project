> 适用：大二在校，11点咖啡店开工型，11408 零基础，晚间有学校项目，夜晚有游戏  
> 时长：2026年6月9日 – 暑假前（约4周）  
> 目标：暑假前能看懂项目核心代码、11408 入门三科、暑假可无缝衔接深度训练

## 一、核心原则

### 第零原则：先稳节奏，再谈效率

你上个月的问题不是"不够努力"，是**只有输出没有回血**。咖啡店学到晚上、项目组干到九点、回去打游戏到凌晨三点——大脑连续高负荷 14 个小时，中间没有一次真正断开。这种状态下效率只会越来越低，最后整个崩掉。

**游戏不是你该戒的东西。** 对你来说，它是紧绷一天后的泄压阀。问题是把它塞在凌晨两点——那不是放松，是扛着疲惫硬撑。应该把它放在项目结束后、上床前，干净利落地玩，然后睡。

### 第一原则：只学一件、学透一件

每天的核心就一件事——**一段代码 + 它背后的理论**。与其泛泛看三个 Controller，不如把一个 `create()` 吃透。

### 第二原则：纸笔优先，电脑验证

咖啡店是个好环境。一杯咖啡、一个笔记本、一支笔——能不开手机的时候绝不开。看懂一段代码→在纸上画出来→再打开电脑验证。画不出来 = 没真懂。

### 第三原则：暂不求甚解

暑假方案要"彻底理解为什么"。学期方案只要"知道是什么、在哪个位置、和谁连接"。**先建地图，再谈细节。**

### 第四原则：弹性伸缩

| 状态 | 当天做法 |
|------|----------|
| 精力好、没杂事 | 按计划执行，多出的时间往后预学或往前复盘 |
| 正常 | 按计划执行 |
| 累/心情不好 | **不勉强**。只做一件事：看一段代码，画个图。然后多玩一小时游戏。明天再来 |
| 几天没碰 | 不补、不自责，从今天直接开始。跳过的周末回看 |

**学习是长期习惯，不是短期冲刺。断几天不可怕，可怕的是因为断了一天就全盘放弃。**

## 三、每日固定节奏

> 按你的真实作息来排。核心改动：**学习块之间强制断开，游戏合法化，12:30 前睡。**

```
11:30 到咖啡店     一杯咖啡，手机扔包里，翻开昨天的笔记看5分钟——昨天最后想了什么，今天接着想。
11:30–13:30        第一个学习块（2h）→ 项目代码回溯
                       中间如果走神了，站起来出去走一分钟，不要硬顶着。
13:30–14:30        午饭 + 散步。不碰手机，不看电脑。
                       去外面走十分钟——这是今天最重要的"回血窗口"。
14:30–16:30        第二个学习块（2h）→ 11408 理论
16:30–17:00        下午茶档：换一杯咖啡/茶，刷点跟学习无关的东西。30分钟纯断开。
17:00–18:30        第三个学习块（1.5h）→ 可选
                       精力好：往后预学或往前深挖今天的内容
                       精力一般：复习今天的笔记，整理手绘图
                       累了：直接跳过，今天就到这里，不算浪费
18:30–19:00        晚饭
19:00–21:00        学校项目组（这是工作，必须做完就断）
21:00–00:30        游戏 / 放松 / 随便干什么（这是你自己的时间，别带着愧疚感玩）
00:30              **上床。手机扔桌子上去。睡。**
```

### 为什么这个节奏不会崩

**三段学习之间各有一个 1 小时以上的真实断开**——午饭散步、下午茶、晚饭。不是"学累了刷五分钟手机"那种假休息。大脑需要的是感官完全换轨：从屏幕到食物、从室内到室外、从文字到身体。

**游戏被放在了正确的位置**——工作结束后，没有负罪感地玩，时间足够尽兴，然后到点睡觉。凌晨三点睡觉不是因为你贪玩，是因为你白天压得太紧、晚上需要大量时间卸载压力。把白天的节奏松动下来，晚上的需求量自然下降。

### 减量信号（出现以下任何一种就立刻降档）

| 信号 | 含义 | 做法 |
|------|------|------|
| 咖啡店坐了一个小时，一行代码没看进去 | 大脑在拒绝 | 合上电脑，去散步。今天只散步，也算过了 |
| 连续两天学到 18:30 之后感到"终于熬完了" | 已经在硬撑 | 明天第三个学习块取消，17:00 直接结束 |
| 游戏时间超过平时的量 | 白天压力没卸掉 | 明天减少一个学习块，多出来的时间做任何非学习的事 |
| 睡前脑子停不下来，反复想代码的事 | 白天缺了"关门仪式" | 睡前最后15分钟写一句话："今天到此为止，剩下的明天继续" |

**一句话记住：你的学习效果不由时长决定，由你清醒时的专注程度决定。4 小时高质量 > 10 小时硬撑。**

## 三-B、关于逃课

你自己评估哪门课值得去。但有一条硬规则：**每周至少去两节你觉得"去了可能有用"的课。** 不是为了成绩，是为了让你的一天有个固定锚点——完全自由安排时间的人，最容易把日子过成一团浆糊。课表上的课就是这个锚。

## 三-C、和家人讨论上线的时机

上线不是纯技术决策。建议不要等项目"完美"再提——**在 Week 2 结束后**，等你画完下单数据流图：

- 拿一张纸把"钱怎么从用户到你手里"讲清楚
- 能回答"出问题了怎么办"（审查报告里至少有 3 个风险点可以引用）
- 不是空口说"我想上线"，而是拿着图说"流程是这样、风险是这些、需要这些支持"

把技术翻译给非技术人员听——这事本身就是重要的能力。

## 九、"看得懂写不出"的学期解法

这个问题要放到暑假集中解决，但本学期可以做铺垫：

### 每周手写练习（安排在周末集中时间）

| 周 | 手写内容 | 目的 |
|----|----------|------|
| W1 | DDL：`order_master` 表的完整建表语句 | 建立 SQL 肌肉记忆 |
| W2 | Java：一个简单的 `public class User { private Long id; }` + getter/setter | 建立 Java 基本语法肌肉记忆 |

**核心方法**：先看一遍原始代码，关掉，纯手写，然后对比差异。差异处就是你下次重点练的地方。
