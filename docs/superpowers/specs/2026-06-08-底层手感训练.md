# 编程底层手感训练

> 目标：从"看得懂"到"写得出来"。  
> 方法：每天一个数据结构/算法，自己从零实现。关 AI。只查 JDK 文档。  
> 然后——回到项目里找到同一个概念被用到的地方，把理论和实践焊在一起。

---

## 为什么你写算法题磕磕绊绊

不是因为刷题少。是因为你缺少"这段代码运行时内存里长什么样"的画面感。  
画不出来就没法 debug，就没法自己写。  
下面每个练习都用纸先画、再写代码、再跑。

---

## Week 1-2：数据结构自己写一遍

| # | 实现什么 | 要求 | 项目里的锚点 |
|---|---------|------|-------------|
| 1 | **动态数组**（类似 ArrayList） | add / get / remove / size，自动扩容。不允许用 `java.util.*` | 项目中所有 `List<VO>` 返回值——底层就是这个 |
| 2 | **单链表** | addHead / addTail / remove / contains / size | RocketMQ 消息就是"队列"，队列用链表实现 |
| 3 | **栈**（用数组实现） | push / pop / peek / isEmpty | JVM 方法调用栈——每调一个方法压一帧 |
| 4 | **队列**（用链表实现） | enqueue / dequeue / peek / isEmpty | 消息队列、请求排队 |
| 5 | **二叉搜索树 BST** | insert / search / delete / 三种遍历（前中后序） | MySQL 索引底层就是 B+Tree，BST 是它的前身 |
| 6 | **哈希表**（数组+链表版本） | put / get / remove / 自动扩容。自己写 hash 函数 | Redis SET/GET 背后就是哈希表 |
| 7 | **堆**（最小堆，用数组实现） | insert / extractMin / peek | 优先级队列、Top K 问题 |
| 8 | **复习**：合上电脑，用纸笔画出上面7个结构的内存布局 | 每个结构的"一个节点长什么样" + "多个节点怎么连" | — |

### 怎么练

```
1. 纸上画出这个数据结构长什么样（15min）
2. 新建一个空 Java 文件，从头写（45min）
   - 不允许复制粘贴
   - 不允许用 AI
   - 可以看 JDK 文档，不能看实现代码
3. 写 main 方法跑几个测试用例（15min）
4. 跑通后，到项目里找到对应的锚点位置，看一遍（15min）
```

## Week 3-4：算法思维

| # | 练什么 | 内容 | 项目里的锚点 |
|---|-------|------|-------------|
| 1 | **排序** | 自己写冒泡、选择、插入、快速排序。不背代码，理解每轮做了什么 | MySQL ORDER BY 可能是 filesort 或者用索引——你建索引就是为了避免全排序 |
| 2 | **二分查找** | 在有序数组里找目标值。写出来之后，改成"找第一个 >= target 的位置" | `redisUtil.set(key, value, ttl)` —— Redis 内部用二分维护过期键 |
| 3 | **树的深度** | 求二叉树最大深度、最小深度、是否平衡 | 项目中 RBAC 角色继承可以建模为树/DAG |
| 4 | **括号匹配** | 用栈判断 `{[()]}` 是否合法 | 前端 JSON 解析、后端 JSON 反序列化 |
| 5 | **LRU 缓存** | 用 HashMap + 双向链表实现 | `ProductCacheService` 就是缓存。Redis 内存淘汰也有 LRU |
| 6 | **Top K** | 用最小堆找数组中第 K 大的数 | 产品列表按销量排序取 Top 10 |
| 7 | **位运算** | 判断奇偶、交换两数、找出只出现一次的数字 | 权限位掩码（`permissions & READ == READ` 那种） |
| 8 | **递归转迭代** | 把二叉树前序遍历用栈重写（不用递归） | 项目中递归不常见，但"递归爆栈"是所有生产环境的问题——理解什么时候用迭代替代 |

### 怎么练

```
1. 读懂题目，在纸上用例子走一遍（10min）
2. 手写代码（30min）
3. 跑通后，解释：时间复杂度和空间复杂度是多少？为什么？（10min）
4. 到项目锚点处看一遍，理解"这个算法在项目中变成了什么"（10min）
```

## 完成标准

四周后找一个 LeetCode 简单题（随机），关掉 AI，20 分钟内 AC。  
不是为了刷题而刷题——是为了验证你从"完全依赖 AI 生成→自己能写"的转变。

---

## 和项目的关系

这个训练跟读项目代码**不冲突**。周一到周五做上面的练习（每天一项），周六日回到项目——那时候你再打开 `OrderServiceImpl.java`，看到 CAS 那行 SQL，你脑子里想的不是"这行代码在干什么"，而是"它在内存里的数据结构是什么，为什么这样写能保证并发安全"。

**底层手感就是你看到代码时脑子里自动浮现的那张内存图。**
