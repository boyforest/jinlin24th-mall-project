## 一、设计原则

### 1.1 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| 表名 | 小写 + 下划线，加前缀区分模块 | `app_user`、`biz_customer` |
| 字段名 | 小写 + 下划线 | `create_time`、`member_level_id` |
| 主键 | 统一用 `id`，bigint 自增 | `id` |
| 索引 | `idx_字段名` 或 `uk_字段名`（唯一索引） | `idx_user_id`、`uk_openid` |

### 1.2 表前缀说明

| 前缀 | 含义 | 示例 |
|------|------|------|
| `app_` | C端小程序相关 | `app_user` |
| `biz_` | B端业务相关 | `biz_customer` |
| `sys_` | 系统配置相关 | `sys_admin`、`sys_role` |
| 无前缀 | 核心业务表 | `product`、`order_master` |

### 1.3 通用字段

每张表都包含以下字段（除特殊说明外）：

```sql
`id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
`create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
```

需要软删除的表增加：

```sql
`deleted` tinyint NOT NULL DEFAULT 0 COMMENT '0-未删除，1-已删除',
```

### 1.4 状态字段设计

- 使用 `tinyint` 类型
- 0 通常表示"初始/未处理/禁用"状态
- 1 通常表示"正常/启用"状态
- 其他值按业务递增（10、20、30...）

## 四、表结构详细设计

### 4.1 用户模块

#### 4.1.1 会员等级表 `member_level`

**用途**：定义会员等级体系，用户根据累计积分自动升级。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 主键 |
| name | varchar(32) | Y | - | 等级名称，如：青铜、白银、黄金 |
| min_points | int | N | 0 | 升级所需最低累计积分 |
| discount | decimal(3,2) | N | 1.00 | 折扣率，1.00=不打折，0.90=9折 |
| icon | varchar(255) | N | NULL | 等级图标URL |
| sort | int | N | 0 | 排序，越小越靠前 |
| status | tinyint | N | 1 | 1-启用，0-禁用 |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**业务规则**：
- 用户累计积分达到 `min_points` 自动升级
- `discount` 字段用于计算会员价：会员价 = 原价 × discount
- 等级按 `sort` 排序显示

**示例数据**：

| id | name | min_points | discount | sort |
|----|------|------------|----------|------|
| 1 | 普通会员 | 0 | 1.00 | 1 |
| 2 | 青铜会员 | 100 | 0.98 | 2 |
| 3 | 白银会员 | 500 | 0.95 | 3 |
| 4 | 黄金会员 | 2000 | 0.90 | 4 |
| 5 | 钻石会员 | 5000 | 0.85 | 5 |

#### 4.1.2 小程序用户表 `app_user`

**用途**：存储小程序端用户信息，支持微信登录。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 用户ID |
| nickname | varchar(64) | N | NULL | 昵称（微信获取） |
| avatar | varchar(255) | N | NULL | 头像URL（微信获取） |
| gender | tinyint | N | 0 | 性别：0-未知，1-男，2-女 |
| openid | varchar(128) | Y | - | 微信openid（当前小程序唯一） |
| unionid | varchar(128) | N | NULL | 微信unionid（跨应用唯一） |
| phone | varchar(20) | N | NULL | 手机号（用户授权后绑定） |
| phone_bind_time | datetime | N | NULL | 手机绑定时间 |
| member_level_id | bigint | N | 0 | 会员等级ID，0=普通用户 |
| points | int | N | 0 | 当前可用积分 |
| total_points | int | N | 0 | 累计获得积分 |
| total_amount | decimal(10,2) | N | 0.00 | 累计消费金额 |
| order_count | int | N | 0 | 订单数量 |
| status | tinyint | N | 1 | 1-正常，0-禁用 |
| deleted | tinyint | Y | 0 | 0-未删除，1-已删除 |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |
| last_login_time | datetime | N | NULL | 最后登录时间 |

**索引设计**：

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |
| uk_openid | 唯一 | openid | openid唯一，用于微信登录 |
| idx_phone | 普通 | phone | 手机号查询 |
| idx_unionid | 普通 | unionid | unionid查询（跨应用场景） |

**业务规则**：
- 用户首次微信登录时，通过 openid 判断是否已存在
- 手机号为可选绑定，绑定后可用于账号合并
- 积分分两种：`points`（当前可用）、`total_points`（累计，用于计算会员等级）
- `total_amount` 和 `order_count` 用于用户画像分析

### 4.2 商品模块

#### 4.2.1 商品分类表 `product_category`

**用途**：商品分类，支持多级分类。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 分类ID |
| parent_id | bigint | N | 0 | 父分类ID，0=顶级分类 |
| name | varchar(64) | Y | - | 分类名称 |
| icon | varchar(255) | N | NULL | 分类图标（小程序首页展示） |
| image | varchar(255) | N | NULL | 分类图片（分类详情页展示） |
| sort | int | N | 0 | 排序 |
| status | tinyint | N | 1 | 1-启用，0-禁用 |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**设计说明**：
- 采用**邻接表模型**（Adjacency List）实现多级分类
- `parent_id = 0` 表示一级分类
- `parent_id > 0` 表示子分类，指向父分类的 id

**示例数据**：

```
一级分类（parent_id=0）
├── 养生茶饮 (id=1)
│   ├── 花草茶 (id=5, parent_id=1)
│   └── 养生茶 (id=6, parent_id=1)
├── 滋补养生 (id=2)
│   ├── 灵芝孢子粉 (id=7, parent_id=2)
│   └── 西洋参 (id=8, parent_id=2)
└── 健康食品 (id=3)
```

#### 4.2.2 商品表 `product`

**用途**：存储商品基本信息。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 商品ID |
| category_id | bigint | Y | - | 分类ID |
| name | varchar(128) | Y | - | 商品名称 |
| subtitle | varchar(255) | N | NULL | 副标题/卖点 |
| main_image | varchar(255) | N | NULL | 主图URL |
| images | text | N | NULL | 图片列表，逗号分隔 |
| video_url | varchar(255) | N | NULL | 视频URL |
| detail | text | N | NULL | 商品详情（富文本HTML） |
| sales | int | N | 0 | 销量（虚拟销量+真实销量） |
| status | tinyint | N | 1 | 1-上架，0-下架 |
| sort | int | N | 0 | 排序 |
| deleted | tinyint | Y | 0 | 0-未删除，1-已删除 |
| create_time | datetime | Y | 当前时间 | 创建时间 |

#### 4.2.3 商品SKU表 `product_sku`

**用途**：存储商品规格信息，管理价格和库存。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | SKU ID |
| product_id | bigint | Y | - | 商品ID |
| sku_name | varchar(128) | Y | - | 规格名称，如：500g/盒 |
| price | decimal(10,2) | Y | - | 售价 |
| member_price | decimal(10,2) | N | NULL | 会员价（为空则无会员价） |
| stock | int | Y | 0 | 库存数量 |
| sku_image | varchar(255) | N | NULL | SKU图片（不同规格可能有不同图片） |
| status | tinyint | N | 1 | 1-启用，0-禁用 |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**设计说明**：
- 一个商品可以有多个 SKU（一对多）
- 用户下单购买的是 SKU，不是商品
- `member_price` 为空时，会员价 = price × member_level.discount
- 库存扣减操作这个表的 `stock` 字段

**示例数据**：

| id | product_id | sku_name | price | member_price | stock |
|----|------------|----------|-------|--------------|-------|
| 1 | 1 | 50g/罐 | 68.00 | NULL | 100 |
| 2 | 1 | 100g/罐 | 128.00 | 118.00 | 50 |
| 3 | 1 | 250g/礼盒 | 298.00 | 268.00 | 20 |

#### 4.3.2 订单主表 `order_master`

**用途**：存储订单基本信息。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 订单ID |
| order_no | varchar(32) | Y | - | 订单编号（唯一） |
| user_id | bigint | Y | - | 用户ID |
| recommender_user_id | bigint | N | NULL | 下单时一级推荐官快照 |
| level2_recommender_user_id | bigint | N | NULL | 下单时二级推荐官快照 |
| total_amount | decimal(10,2) | Y | - | 订单总金额 |
| pay_amount | decimal(10,2) | Y | - | 实付金额 |
| freight_amount | decimal(10,2) | N | 0.00 | 运费 |
| discount_amount | decimal(10,2) | N | 0.00 | 优惠金额 |
| points_used | int | N | 0 | 使用积分 |
| points_gained | int | N | 0 | 获得积分 |
| status | tinyint | Y | 0 | 订单状态（见下方说明） |
| pay_type | tinyint | N | NULL | 支付方式：1-微信支付 |
| pay_time | datetime | N | NULL | 支付时间 |
| delivery_time | datetime | N | NULL | 发货时间 |
| receive_time | datetime | N | NULL | 收货时间 |
| receiver_name | varchar(64) | Y | - | 收货人 |
| receiver_phone | varchar(20) | Y | - | 收货电话 |
| receiver_address | varchar(255) | Y | - | 收货地址 |
| remark | varchar(255) | N | NULL | 订单备注 |
| admin_id | bigint | N | NULL | 处理的管理员ID |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**订单状态说明**：

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 待付款 | 订单创建，等待用户支付 |
| 10 | 待发货 | 用户已支付，等待商家发货 |
| 20 | 待收货 | 商家已发货，等待用户确认 |
| 30 | 已完成 | 用户已确认收货 |
| 40 | 已取消 | 用户取消或超时未支付 |
| 50 | 退款中 | 用户申请退款 |
| 60 | 已退款 | 退款完成 |

**状态流转**：

```
待付款(0) ──支付──► 待发货(10) ──发货──► 待收货(20) ──确认──► 已完成(30)
    │                                                    ▲
    │                                                    │
    └──取消──► 已取消(40)                                 │
                                                         │
                        待发货(10) ──退款──► 退款中(50) ──完成──► 已退款(60)
```

**订单编号生成规则**：
```
格式：JL + 年月日时分秒 + 4位随机数
示例：JL202604021530451234
```

#### 4.3.3 订单明细表 `order_item`

**用途**：存储订单商品明细（快照）。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 明细ID |
| order_id | bigint | Y | - | 订单ID |
| order_no | varchar(32) | Y | - | 订单编号 |
| product_id | bigint | Y | - | 商品ID |
| sku_id | bigint | Y | - | SKU ID |
| product_name | varchar(128) | Y | - | 商品名称（快照） |
| sku_name | varchar(128) | N | NULL | 规格名称（快照） |
| product_image | varchar(255) | N | NULL | 商品图片（快照） |
| price | decimal(10,2) | Y | - | 单价（快照） |
| quantity | int | Y | - | 数量 |
| total_price | decimal(10,2) | Y | - | 小计 |
| create_time | datetime | Y | 当前时间 | 创建时间 |

**为什么需要快照？**

> 商品信息可能会变（改名、改价、下架），但历史订单不能变。
> 
> 用户下单时把商品信息**复制**一份存到订单明细表，之后商品怎么变都不影响历史订单。

**示例**：

用户下单时商品信息：
- 商品：黄山毛峰茶
- 规格：100g/罐
- 价格：¥128

三个月后商家改价为 ¥158，但用户查看历史订单时，显示的仍然是 ¥128。

### 4.4 营销模块

#### 4.4.1 优惠券表 `coupon`

**用途**：优惠券模板，定义优惠券规则。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 优惠券ID |
| name | varchar(64) | Y | - | 优惠券名称 |
| type | tinyint | Y | 1 | 类型：1-满减券，2-折扣券，3-固定金额券 |
| min_amount | decimal(10,2) | N | 0.00 | 使用门槛金额 |
| discount_value | decimal(10,2) | Y | - | 优惠值 |
| stock | int | Y | 0 | 发放总量 |
| received_count | int | N | 0 | 已领取数量 |
| used_count | int | N | 0 | 已使用数量 |
| start_time | datetime | Y | - | 生效时间 |
| end_time | datetime | Y | - | 过期时间 |
| member_level_id | bigint | N | NULL | 限会员等级，NULL=不限 |
| status | tinyint | N | 1 | 1-启用，0-禁用 |
| create_time | datetime | Y | 当前时间 | 创建时间 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**优惠券类型说明**：

| 类型 | 说明 | discount_value 含义 |
|------|------|---------------------|
| 1-满减券 | 满 X 减 Y | Y = 减免金额 |
| 2-折扣券 | 打折 | 90 = 9折，85 = 8.5折 |
| 3-固定金额券 | 直接减免 | 减免金额 |

**示例**：

| name | type | min_amount | discount_value | 说明 |
|------|------|------------|----------------|------|
| 满200减20 | 1 | 200.00 | 20.00 | 订单满200减20 |
| 新人9折券 | 2 | 0.00 | 90.00 | 无门槛，打9折 |
| 5元无门槛券 | 3 | 0.00 | 5.00 | 直接减5元 |

#### 4.4.2 用户优惠券表 `user_coupon`

**用途**：用户领取的优惠券。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | ID |
| user_id | bigint | Y | - | 用户ID |
| coupon_id | bigint | Y | - | 优惠券ID |
| order_id | bigint | N | NULL | 使用的订单ID |
| status | tinyint | Y | 0 | 状态：0-未使用，1-已使用，2-已过期 |

#### 4.5.2 库存表 `inventory`

**用途**：记录每个 SKU 在每个仓库的库存。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 库存ID |
| warehouse_id | bigint | Y | - | 仓库ID |
| sku_id | bigint | Y | - | SKU ID |
| stock | int | Y | 0 | 当前库存 |
| warning_stock | int | N | 10 | 库存预警值 |
| update_time | datetime | Y | 当前时间 | 更新时间 |

**设计说明**：
- 联合唯一索引 `(warehouse_id, sku_id)` 保证一个 SKU 在一个仓库只有一条库存记录
- 库存预警：当 `stock < warning_stock` 时提醒补货

#### 4.5.3 库存流水表 `inventory_log`

**用途**：记录所有库存变动，可追溯。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | Y | 自增 | 流水ID |
| warehouse_id | bigint | Y | - | 仓库ID |
| sku_id | bigint | Y | - | SKU ID |
| type | tinyint | Y | - | 类型：1-入库，2-出库，3-盘点调整 |
| quantity | int | Y | - | 变动数量（正=入，负=出） |
| before_stock | int | Y | - | 变动前库存 |
| after_stock | int | Y | - | 变动后库存 |
| order_no | varchar(32) | N | NULL | 关联订单号 |
| remark | varchar(255) | N | NULL | 备注 |
| operator_id | bigint | N | NULL | 操作人ID |
| create_time | datetime | Y | 当前时间 | 操作时间 |

**设计说明**：
- 每次库存变动都记录流水，保证可追溯
- `before_stock` + `quantity` = `after_stock`，用于校验数据一致性
- 出库时 `quantity` 为负数

## 五、索引设计原则

### 5.1 什么时候建索引

| 场景 | 是否建索引 |
|------|------------|
| 主键 | ✅ 自动建主键索引 |
| 外键关联字段 | ✅ 建普通索引 |
| 唯一约束字段 | ✅ 建唯一索引 |
| WHERE 条件常用字段 | ✅ 建普通索引 |
| ORDER BY 排序字段 | ⚠️ 看情况 |
| 很少查询的字段 | ❌ 不建 |
| 数据量小的表 | ❌ 不建 |

### 5.2 索引命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 主键索引 | PRIMARY | 自动 |
| 唯一索引 | uk_字段名 | uk_openid |
| 普通索引 | idx_字段名 | idx_user_id |
| 联合索引 | idx_字段1_字段2 | idx_warehouse_sku |

## 八、注意事项

1. **金额字段统一用 decimal(10,2)**，不要用 float/double，避免精度问题
2. **时间字段统一用 datetime**，不要用 timestamp（范围限制）
3. **状态字段用 tinyint**，便于扩展
4. **软删除字段 deleted**：重要的业务数据不要物理删除
5. **字段注释必写**：方便后续维护
6. **索引不要滥用**：索引会降低写入性能
